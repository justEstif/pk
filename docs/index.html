works with your other tools

Claude bridges your sources.
pk saves what matters.

If you've connected Slack, Confluence, Granola, or other tools to Claude alongside pk, you can ask Claude to pull relevant context from them and save it to pk in one conversation.

pk doesn't orchestrate any of this automatically — Claude does the coordination when you ask it to. pk is just where the knowledge lands and persists across every future session.

mcp tools

One knowledge base.
Works wherever you use AI.

Connect pk to Claude Desktop, Cursor, or any AI tool that supports MCP. Then just ask — Claude searches your notes, summarizes what you know, or captures something new.

pk_search

Ask and retrieve

Full-text search across everything you've captured. Ask Claude "what did I decide about the database?" and get ranked results with context.

pk search "postgres" --type decision

pk_synthesize

Catch me up

Summarise everything you know about a topic, or the whole project at once. Start a new session with full context instead of starting blank.

pk synthesize "auth decisions" --all

pk_new

Capture as you go

Tell Claude to save a decision, a question, or a note. It writes it in a structured format so you — and future Claude sessions — can find it later.

pk new decision "Drop Redis" --tags arch

pk_lint

Keep it clean

Validates every note for completeness. Flags stale questions, decisions missing rationale, and anything that's drifted out of shape.

pk lint # validates schema + quality

knowledge model

Five note types.
Not a flat dump.

A typed schema lets agents query with precision. Notes live in ~/.pk/ as plain markdown — readable without any tool.

note

Durable project knowledge — facts, patterns, context worth keeping.

decision

Chosen direction with rationale and consequences. Answers "why did we do this?"

question

Unresolved uncertainty blocking or informing the next move.

source

Raw input with provenance — the meeting notes, the Slack thread, the doc.

index

Navigation map over a topic. Generated by pk rebuild.

setup

1

Install & init

Run pk init myproject --harness claude. Creates a knowledge home for your project and registers pk with Claude Desktop automatically.

2

Agent discovers tools

Open Claude Desktop and pk is already available. No extra steps. Just start a conversation and Claude can read and write your knowledge base.

3

Knowledge accumulates

Agents write decisions and sources while they work. Plain markdown. Git-diffable. Readable without any tool.

4

Switch projects in seconds

New project, new session. The agent searches the right base. Context loads in a tool call, not 20 minutes of re-explaining.
